Описание таблиц БД.

В проекте используются две таблицы:
  users - информация о пользователе,
  tasks - информация о задаче. 

Таблица USERS

Состоит из следующих столбцов: 
  id - уникальный код пользователя, указывается автоматически; 
  name - имя пользователя, обязательное поле; 
  username - уникальный идентификатор при регистрации, обязательное поле; 
  password - пароль для регистрации и дальнейшего входа, обязательное поле; 
  created_time - время регистрации, указывается автоматически. 

Таблица TASKS

Состоит из столбцов: 
  user_id - уникальный код пользователя;
  id - уникальный код задачи, указывается автоматически; 
  name - название задачи, обязательное поле;
  description - описание задачи, может быть пустым;
  status - статус выполнения задачи, выполнено (или 1 в таблице) / не выполнено (или 0 в таблице), по умолчанию не выполено (0);
  limit_time - время, до которого нужно сделать задачу, может быть пустым;
  created_time - время создания задачи, указывается автоматически; 
  priority - приоритет выполнения задачи, низкий (или 0 в таблице) / средний (или 1 в таблице) / высокий (или 2 в таблице), по умолчанию низкий (0).

Связь между таблицами осуществляется через столец id в таблице users и столбец user_id в таблице tasks. Таким обрахом пользователь види только свои задачи и никакие другие.
