Описание таблиц БД.

В проекте используются две таблицы:
  users - информация о пользователе, tasks - информация о задаче. 

Таблица USERS

Состоит из следующих столбцов: 
  - _id_ - уникальный код пользователя, указывается автоматически; 
  - _name_ - имя пользователя, обязательное поле; 
  - _username_ - уникальный идентификатор при регистрации, обязательное поле; 
  - _password_ - пароль для регистрации и дальнейшего входа, обязательное поле;
  - _created_time_ - время регистрации, указывается автоматически. 

Таблица TASKS

Состоит из следующих столбцов: 
  - _user_id_ - уникальный код пользователя;
  - _id_ - уникальный код задачи, указывается автоматически; 
  - _name_ - название задачи, обязательное поле;
  - _description_ - описание задачи, может быть пустым;
  - _status_ - статус выполнения задачи, выполнено (или 1 в таблице) / не выполнено (или 0 в таблице), по умолчанию не выполено (0);
  - _limit_time_ - время, до которого нужно сделать задачу, может быть пустым;
  - _created_time_ - время создания задачи, указывается автоматически; 
  - _priority_ - приоритет выполнения задачи, низкий (или 0 в таблице) / средний (или 1 в таблице) / высокий (или 2 в таблице), по умолчанию низкий (0).

Связь между таблицами осуществляется через столбец id в таблице users и столбец user_id в таблице tasks. Таким обрахом пользователь види только свои задачи и никакие другие.
